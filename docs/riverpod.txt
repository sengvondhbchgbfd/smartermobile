Riverpod is a modern state management library for Flutter created by Rémi Rousselet (same creator of Provider).
It helps you manage state safely, cleanly, and efficiently.


Why use Riverpod? |=> 
Compared to basic setState():
❌ setState() → messy in large apps
❌ hard to manage data flow
❌ causes unnecessary rebuilds
✅ Riverpod → clean architecture
✅ better performance
✅ testable & scalable
✅ no BuildContext needed

////////////////////////


Riverpod works with Providers.
👉 A Provider = a place that stores and gives data

///////////////////////////

Since you are building systems (like your Gym system 👇), Riverpod is great for:
Authentication
API data
Dashboard state
Real-time updates

//////////////////////////////////////////////////////////////


flutter packages upgrade  |=> Upgrade packages
dart run build_runner clean |=>  Clean old files
flutter pub get |=> Get packages
dart run build_runner build --delete-conflicting-outputs  |=> Generate
dart run build_runner build --delete-conflicting-outputs |=> Every time you change a provider file, just run
dart run build_runner watch --delete-conflicting-outputs |=>  Or auto watch mode (run once, auto generates on save)
